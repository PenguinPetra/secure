# セキュア認証アプリケーション 

---

## 1. プロジェクト概要
本プロジェクトは、Next.js 15 を使用した「ガチガチにセキュアな設計」をコンセプトとする、セッションベースの認証アプリケーションです。

---

## 2. 実行環境・技術スタック
*   **フレームワーク**: Next.js 15 (App Router / TypeScript) 
*   **データベース**: SQLite (Prisma 6系) 
*   **認証方式**: セッションベース認証 (Cookie管理) 
*   **主要ライブラリ**:
    *   **bcrypt**: パスワードの安全なハッシュ化 (ソルト付き)
    *   **swr**: CAPTCHAデータの効率的な取得と更新（連鎖レンダリング防止） 
    *   **lucide-react**: セキュリティ意識を高める視覚的UIアイコン 

---

## 3. 実装した独自機能

### ① パスワードの強度機能
*   **パスワード強度判定 & リアルタイムアドバイス**: 長さ（8文字以上必須）と文字種（大文字・数字・記号）をリアルタイムで評価。脆弱な場合は登録をブロックし、具体的な改善アドバイスをユーザーに提示します。
*   **パスワード表示切り替え**: 入力ミスを防ぐため、目のアイコンで平文と伏せ字を切り替え可能です。

*   **二重入力チェック**: 入力ミスによる「パスワードを忘れた」状態を防ぐため、厳格な一致チェックを行っています。不一致の状態ではアカウント作成ボタンを無効化する防御的設計です。

### ③ 防御：Bot・ブルートフォース攻撃対策
*   **CAPTCHA**: HMAC署名を用いた計算クイズ。Botによる自動送信を物理的に遮断します。(左図の画像)
*   **レートリミット（ログイン試行制限）**: 同一IPからの「パスワードミス」および「CAPTCHAミス」を合計して監視。**15分以内に5回失敗**した場合、システムが自動的に15分間のアクセスロックをかけ、総当たり攻撃を無効化します。(右図の画像)

### ➃ セッション監視と遠隔ログアウト（監視・制御）
*   **アクティブセッションの可視化**: 現在ログイン中の全端末のIPアドレスとUser-Agentをダッシュボードに表示。ユーザー自身が「身に覚えのないログイン」を即座に検知できます。
*   **セッション破棄**: 他端末のセッションをDBから直接削除することで、外出先でのログアウト忘れや端末紛失時のリスクを最小限に抑えます。

## 4. セキュリティ上の工夫（ガチガチな設計）
*   **パスワードのハッシュ化**: **bcrypt**を使用し、計算負荷（コストファクタ）を調整することで、GPUによる高速な総当たり攻撃やレインボーテーブル攻撃を無効化しています。(上図の画像)
*   **セキュアCookie属性の徹底**: ブラウザの開発者ツールで確認可能な通り、以下の属性を付与しています。（下図の画像）
    *   **HttpOnly**: JavaScriptからのアクセスを禁止し、XSSによるトークン窃取を防止。
    *   **SameSite=Strict**: 外部サイトからのCookie送信を完全に制限し、CSRF攻撃を遮断。
*   **厳格なCSP設定**: `next.config.ts`にて**Content Security Policy** を定義。インラインスクリプトや不正な外部リソースの読み込みをブラウザレベルで制限しています。
